# Quest Taxonomy and Naming Rules

## The problem this fixes

Students should not need to translate lore names such as “Evidence Royale” to figure out that an activity is actually multiple choice. A task name must make the academic format unmistakable.

## Required card structure

```text
[ACTIVITY TYPE]: [Historical Topic]
[Mode] • [Estimated time] • [Question / source count]
Builds: [Badge name(s)]
Rewards: +[XP] Historian XP • +[Tokens] Archive Tokens
```

## Approved activity types

Use these as the large, visible first line on all cards and buttons:

- MCQ Challenge
- Vocabulary Challenge
- HIPP Source Analysis
- Primary Source Analysis
- Timeline Challenge
- SAQ Practice
- DBQ Writing Challenge
- LEQ Writing Challenge
- Debate Challenge
- Review Challenge
- Reflection / Revision Challenge

## Approved modes

Use these as a smaller secondary tag:

- Practice
- Core Quest
- Side Quest
- Collaborative Quest
- Boss Battle
- Bonus Challenge

Boss Battle is a mode—not a complete label by itself.

### Good examples

```text
MCQ Challenge: Columbian Exchange
Core Quest • 12 questions • 10 minutes
Builds: MCQ Mastery Badge, Exchange Investigator Badge
Rewards: +30 XP • +3 Archive Tokens
```

```text
DBQ Writing Challenge: Building Colonial America
Boss Battle • 60 minutes • 7 documents
Builds: DBQ Mastery Badge, Foundations of America Seal
Rewards: +60 XP • +6 Archive Tokens
```

```text
HIPP Source Analysis: John Winthrop’s “A Model of Christian Charity”
Practice • 1 source • 12 minutes
Builds: HIPP Mastery Badge, Colonial Sourcekeeper Badge
Rewards: +20 XP • +2 Archive Tokens
```

### Names to retire

| Retire | Replace with |
|---|---|
| Evidence Royale | MCQ Challenge: [Topic] |
| Primary Source Battle | Primary Source Analysis: [Topic] |
| Boss Battle | [Activity Type]: [Topic] + Boss Battle mode |
| Debate Duel | Debate Challenge: [Topic] |
| Timeline Mission | Timeline Challenge: [Topic] |

“Bounty,” “royale,” “duel,” and similar language can still appear in map lore, an illustration caption, or a subheading. They should not replace the visible activity type.
